# МаркетологТЕХ — лендинг и демо-дашборд

Сервис возврата клиентов для салонов красоты на YClients.
Чистый HTML/CSS/JS, без сборки и зависимостей.

## Как открыть

Двойным кликом по `index.html` — всё заработает.

Если хотите как на живом хостинге (относительные пути, кеш, без ограничений `file://`):

```bash
cd "путь/к/папке"
python3 -m http.server 4173
# затем откройте http://127.0.0.1:4173
```

## Файлы

```
index.html            Лендинг (9 блоков брифа)
demo.html             Демо-дашборд, 5 экранов, read-only
assets/css/base.css   Дизайн-система: токены обеих тем, стекло, типографика, кнопки
assets/css/landing.css Стили лендинга
assets/css/demo.css   Стили дашборда
assets/js/landing.js  Валидация формы, отправка, появление секций
assets/js/demo.js     Переключение экранов и фильтров
assets/js/theme.js    Переключатель светлой/тёмной темы
```

## Что нужно заменить перед запуском

| Что | Где | Сейчас |
|---|---|---|
| Срок ответа `[N]` | `index.html`, поиск по «в течение 24 часов» | 24 часа |
| Салонов в пилоте `[N]` | `index.html`, блок «Доверие» | 10 салонов |
| Логотип | `.logo__mark` в `index.html` и `demo.html` | буква «М» в фиолетовом квадрате |
| Точный HEX акцента | `--accent` в `assets/css/base.css` | `#6B5BFF` |
| Первый кейс клиента | `index.html`, карточка `.card--empty` | заглушка «Скоро здесь» |

Логотип: замените `<span class="logo__mark">М</span>` на `<img src="assets/logo.svg" alt="">`.
Место одно и то же в обоих файлах.

## Темы

Сайт открывается в **светлой** теме. Кнопка в шапке (луна/солнце) переключает
на тёмную, выбор запоминается в `localStorage` и переживает перезагрузку.

Обе темы описаны токенами в начале `base.css`: светлая в `:root`, тёмная
в `:root[data-theme="dark"]`. В компонентах захардкоженных цветов нет —
если добавите новый блок, берите цвет из переменной, иначе он не переключится.

Ключевой приём — токены `--contrast-*`. Это карточка, **противоположная** фону:
в тёмной теме белая, в светлой почти чёрная. Поэтому класс называется
`contrast-card`, а не «белая карточка» — иначе в светлой теме имя врало бы.

Тема применяется маленьким скриптом прямо в `<head>` обеих страниц, до отрисовки.
Без него тёмная тема моргала бы белым при загрузке. Не выносите его в отдельный
файл — задержка загрузки вернёт мигание.

Контраст текста проверен: все основные пары проходят WCAG AA (4.5:1)
в обеих темах.

## Форма заявки

Сейчас это **заглушка**: заявка валидируется и пишется в консоль браузера,
пользователь видит экран «Заявка принята».

Чтобы подключить реальный приём — замените тело `submitLead()` в
`assets/js/landing.js` (там же лежит готовый пример с `fetch`).

⚠️ Токен Telegram-бота нельзя класть в этот файл: всё, что попадает во фронтенд,
видно любому, кто откроет исходники страницы. Нужна прослойка на сервере
(serverless-функция), которая держит токен у себя и уже она пишет в Telegram.

Форм две, обе шлют в `submitLead()` и различаются полем `source`:
`landing:form` (блок 6) и `landing:final-cta` (блок 9).

## Решения по вёрстке, которые стоит знать

**Порядок секций.** В разметке секции лежат в мобильном порядке из брифа
(форма сразу после «Как работает»). На экранах от 900px CSS-свойство `order`
переставляет форму на её место по нумерации блоков — после «Почему YClients».
Разметка одна, дублирования нет. Правила — в конце `landing.css`.

**Фон.** Градиент с боке-пятнами вместо фотографии: стекло на нём читается
чище, и не нужно тащить тяжёлую картинку. Цвета берутся из токенов темы.
Если захотите фото — подмените `background` у `.bg` в `base.css`, но тогда
понадобятся две картинки, по одной на тему.

**Демо read-only.** Кнопки действий («Отправить», «Опубликовать», «Запустить»)
намеренно не работают — это витрина. Живые только переключение вкладок
и фильтры ленты на экране «Сегодня».

**Данные в демо вымышленные** и об этом честно сказано плашкой сверху.

## Доступность

Вкладки дашборда — с ролью `tablist`, переключаются стрелками ←/→.
Формы размечены `label`, ошибки проставляют `aria-invalid`.
Анимации отключаются при `prefers-reduced-motion`.
