# МаркетологТЕХ — лендинг геомаркетинга для салонов

Статический прототип продающего лендинга и четырёхшаговой воронки бесплатного
аудита. Сборка и внешние зависимости не нужны.

## Что внутри

- `index.html` — новый лендинг и разметка воронки;
- `assets/css/landing.css` — самостоятельная светлая дизайн-система лендинга;
- `assets/js/landing.js` — шаги аудита, CRM-онбординг, выбор тарифа и UI-состояния;
- `assets/img/salon-audit-cover.jpg` — демонстрационная фотография салона;
- `demo.html` — интерактивный прототип рабочего продукта: задачи, направления и отчёты;

## Локальный просмотр

```bash
python3 -m http.server 4173
```

Затем открыть `http://127.0.0.1:4173/index.html`.

## Воронка аудита

1. Посетитель вводит название салона.
2. Подтверждает найденную организацию.
3. Выбирает YClients, другую CRM или вариант без CRM и указывает число филиалов.
4. Выбирает Telegram или MAX, оставляет контакт и получает краткий демонстрационный результат.
5. Интерфейс показывает все три тарифа и отмечает подходящий по ответам посетителя.

Данные отчёта сейчас вымышлены и явно подписаны как демонстрационные. Позиция в
поиске и расчёт потенциальной выручки намеренно не используются.

## Точки подключения для программиста

### 1. Поиск организации

Сейчас `openFlow()` в `assets/js/landing.js` подставляет введённое название в
демонстрационную карточку. Нужно заменить это на поиск по прямым ссылкам Яндекс
Карт и 2ГИС или на серверный поиск организации.

### 2. CRM

- `flow-crm=yclients` — запуск OAuth YClients;
- `flow-crm=other` — индивидуальный сценарий подключения;
- `flow-crm=none` — только геомаркетинг.

### 3. Реальный аудит

Экран с классом `.scan-progress-card` сейчас показывает короткую UI-анимацию.
Её нужно связать со статусами серверной задачи. Краткий результат заполняется в
блоке `[data-flow-step="4"]`.

### 4. Мессенджер и заявка

Точка будущего запроса отмечена комментарием внутри обработчика
`flow-contact-form` в `assets/js/landing.js`. Форма передаёт выбранный канал в
поле `messenger`: для Telegram контакт вводится как ник, для MAX — как номер
телефона. Секреты и токены ботов нельзя размещать во фронтенде — форма должна
обращаться к серверному API.

Для MAX номер в форме является контактным и проверочным полем, но не адресом
для Bot API. Пользователь должен открыть бота и нажать «Начать», после чего
сервер сохранит его `user_id` из события MAX и сможет отправить готовый отчёт.

### 5. Тарифы

Предварительная логика находится в `setRecommendedTariff()`:

- без CRM — «Гео», 7 000 ₽;
- с CRM — «Гео + CRM», 12 500 ₽;
- больше пяти филиалов — «Сеть», индивидуально.

## Замена фотографии и логотипа салона

Фотография используется в трёх местах через один файл
`assets/img/salon-audit-cover.jpg`. В реальном сценарии источник изображения и
буква в `.salon-logo`, `.found-logo`, `.result-logo` должны заполняться данными
организации. Если логотипа нет, можно оставлять первую букву названия.

## Важно перед публикацией

- подключить реальный поиск салона;
- подключить API аудита и ботов Telegram/MAX;
- заменить демонстрационные показатели реальными;
- подтвердить финальные тарифы и условия раннего доступа;
- добавить аналитику событий воронки и политику обработки данных.
